# Grille d’évaluation – TP Observabilité (Logs + Metrics + Health)

**Travail en binôme**

Doit être rendu sous la forme d’un **repository accompagné d’un README structuré**.

La note finale est calculée sur **100 points**.

---

# 1. Fonctionnement global du projet (30 points)

| Critère | Description |
|--------|-------------|
| Application fonctionnelle | L’application fonctionne et répond correctement aux requêtes |
| Observabilité opérationnelle | Les mécanismes mis en place permettent d’observer le comportement de l’application |
| Cohérence globale | Les différentes parties (logs, métriques, health) sont cohérentes entre elles |
| Stabilité du projet | Le projet fonctionne de manière reproductible sans comportement inattendu |

---

# 2. Implémentation technique (30 points)

| Critère | Description |
|--------|-------------|
| Qualité du logging | Les logs sont structurés, exploitables et correctement utilisés |
| Pertinence des métriques | Les métriques permettent d’observer efficacement le comportement de l’application |
| Gestion des états du service | La distinction entre les différents états du service est correctement implémentée |
| Intégration dans le cycle de vie | L’instrumentation est correctement intégrée au fonctionnement réel de l’application |

---

# 3. Reproductibilité et qualité technique (10 points)

| Critère | Description |
|--------|-------------|
| Lancement du projet | Le projet peut être lancé facilement en suivant les instructions |
| Simplicité d’exécution | Le projet ne nécessite pas de manipulation complexe ou non documentée |

---

# 4. README et documentation (20 points)

| Critère | Description |
|--------|-------------|
| Notice d’installation | Instructions claires pour lancer le projet |
| Guide d’utilisation | Étapes permettant d’observer les comportements attendus |
| Réponses théoriques | Réponses structurées et compréhensibles |
| Observations | Preuves concrètes (logs, outputs, tests) des comportements observés |

---

# 5. Compréhension et analyse (10 points)

| Critère | Description |
|--------|-------------|
| Compréhension des concepts | Les explications montrent une bonne compréhension de l’observabilité |
| Analyse des résultats | Les comportements observés sont expliqués et interprétés correctement |

---

# Pénalités possibles

| Situation | Pénalité |
|----------|----------|
| Projet impossible à lancer | -20 |
| README absent ou incomplet | -15 |
| Observabilité non fonctionnelle | -20 |

---

# Barème final

| Note | Interprétation |
|------|--------------|
| 90–100 | Travail excellent, compréhension solide |
| 75–89 | Travail maîtrisé |
| 60–74 | Compréhension partielle |
| <60 | Objectifs non atteints |
